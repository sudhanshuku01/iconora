// usage

Simple API

Two ways to use Iconora. Named imports are recommended for optimal tree-shaking.

// Named import — best tree-shaking
import {
  HomeIcon,
  SearchIcon,
  BellIcon
} from '@iconora/react';

export function Nav() {
  return (
    <nav>
      <HomeIcon size={24} />
      <SearchIcon
        size={20}
        color="#3b82f6"
      />
      <BellIcon
        size={20}
        strokeWidth={1.5}
        className="text-gray-500"
      />
    </nav>
  );
}
// Generic <Icon> — dynamic names
import { Icon } from '@iconora/react';

<Icon name="home" size={20} />
<Icon
  name="heart"
  size={20}
  color="red"
  label="Liked"
/>

// why iconora

Built Right.

Every decision made with production in mind.

Tree-shakeable

Import only the icons you use. sideEffects: false and proper ESM exports ensure zero dead code in your bundle.

Accessible by default

Decorative icons get aria-hidden. Pass a label prop and get aria-label + role="img" automatically.

TypeScript first

Full type inference on every prop, generated .d.ts files, and autocomplete in every editor out of the box.

Framework-agnostic core

@iconora/core ships pure SVG path data. Build adapters for Vue, Svelte, Angular, or any framework you need.

Consistent design

Every icon sits on a 24×24 grid with identical stroke widths, cap styles, and optical weight. Nothing looks out of place.

Open source

MIT licensed. Use it anywhere, modify freely, contribute back. Built in the open on GitHub with automated releases via Changesets.
